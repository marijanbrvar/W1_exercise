Week 1 excersises showcase

In this exercise, we are adding a form for posting comments to the article. Our goal here is to master HTML forms and play a little with UX and fonts. You learned about all of these in previous steps. Having fun!

Task:

  1. Create a new branch in the repository of your previous project.
  2. First of all, think about the user experience:
    1. Where should this form be placed?
    2. Should it be fully visible or should it be collapsed until a user decides to open it?
    3. Would it be useful to have multiple links in different parts of the article that can move a user to thecomment’s form quickly or would it be too aggressive?
  3. Make your own decisions about UX but remember to describe them in your PR description, so your code reviewer can easily understand your reasons.
  4. Secondly, create a nice form that includes:
    1. a short text field for user’s name
    2. a short text field for email
    3. a long text field for comment
    4. a question: “What is your overall opinion about this article” with 3 options positive/negative/neutral
    5. only one of the above options can be selected
    6. a checkbox saying “I want to subscribe to the newsletter”
    7. a submit button saying “Send”
  5. Add a placeholder for the comment field saying “Type your comment here…”
  6. Add validations that will:
    1. make sure that name, email and comment fields are required
    2. make sure that comment field can have up to 250 chars and other fields can have up to 20 chars

What do you think?
